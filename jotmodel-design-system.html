The one decision that shapes everything

Two layers of color, never mixed

In a modeling tool, color is the user's vocabulary. If the brand spends color on decoration, it competes with the meaning people assign when they color-code a domain. So JotModel keeps almost no color in its chrome — and reserves saturated color entirely for the canvas. The result: if something on the board is colored, a person made it so.

The rule that makes it obvious

Interface color only ever appears on chrome you click. Semantic color only ever appears as a fill on objects you made. Same eye, two jobs — never the same place. That's how color-coding stays unmistakable without a legend.

Build spec

Principles

  1. Chrome is quiet; color means the user said so. The app never decorates with color — neutrals + one accent for interaction, nothing more.
  2. Type-first, mouse-equal. Create and connect from keyboard or direct manipulation; neither is a fallback.
  3. No galleries, no drawers. Structure is entered inline, never configured in a panel that steals attention.
  4. Loose in the room, strict underneath. Fields stay loose during discussion; the model exports clean DBML / dbt / SQL.
  5. Light and dark are equals. Every token has both; nothing is designed for one and patched for the other.
