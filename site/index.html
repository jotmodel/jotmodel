How it works

Type an entity. Drag to relate. Done.

The whole tool is built around one motion. You write the way you talk, and JotModel keeps the diagram tidy — no syntax, no conventions, no code required.

  1. 1

    Type an entity

    Click anywhere, name the table, and list fields in plain English — comma-separated.

  2. 2

    Types infer themselves

    JotModel reads each field name and assigns a sensible type. Wrong guess? Override it inline.

  3. 3

    Drag to relate

    Pull from a table’s edge to another table. The relationship draws itself, with cardinality.

You type JotModel reads
  • order date→date
  • active?→boolean
  • amount→number
  • email→email
  • id→pk
  • created at→timestamp

Snake_case is supported — never assumed.

Hand off clean

A whiteboard on top. Production-clean schema underneath.

Fields stay loose plain-English while the conversation is live. The moment you need real artifacts, the same model exports clean — no cleanup pass. The messy notes are the model.

  • DBML Portable schema, ready for dbdiagram and friends.
  • dbt A schema.yml plus model stubs to drop in.
  • SQL DDL Postgres, MySQL, or SQL Server — pick your dialect.
model.dbml 
Table users {
  id          integer   [pk]
  email       varchar
  created_at  timestamp
}

Table orders {
  id          integer   [pk]
  user_id     integer   [ref: > users.id]
  amount      decimal
  order_date  date
}

// one user, many orders
Ref: orders.user_id > users.id

FAQ

Questions, answered.

Do I need to know SQL?

No. You type in plain English — “order date”, “amount”, “active?” — and JotModel infers a sensible type for each field. You never have to write SQL or follow naming conventions, and you can override any inferred type inline.

I’m a PM or business analyst — can I use this?

Yes — that’s exactly who it’s for. JotModel reads plain English, so a PM, analyst, or domain expert can lead the modeling session instead of watching one engineer type. The whole room shapes the model together.

What can I export?

DBML (portable and dbdiagram-ready), dbt (a schema.yml plus model stubs), and SQL DDL for Postgres, MySQL, and SQL Server. The export is production-clean — no cleanup pass required. (Importing an existing schema is on the roadmap.)

Does it work offline?

Yes. JotModel is local-first: boards load instantly and keep working offline — even on conference-room wifi. You sync to the cloud when you want to, not before.

Is it really free?

Yes. The Free plan gives you 10 boards forever with the full app — plain-English input, all exports, and real-time collaboration — with no credit card. Paid plans for unlimited boards and team features are coming soon.

How do accounts work?

Purely local boards need no account at all. To save to the cloud or share a board, you create a free account with an email and password. More sign-in options are coming.
